# Day 3 — Bigram Language Model: From Counts to a Neural Net

## Q1: Turning the "count matrix" into a "probability matrix"

```python
P = (N+1).float()
P /= P.sum(1, keepdims=True)
```

These are the two lines from Karpathy's makemore Part 1 that convert the **count matrix** into a **probability matrix**. Let's go line by line.

### First, recall what `N` is

`N` is a **27×27 integer matrix** (26 letters + 1 special symbol `.`). It records the bigram counts in the training set:

> `N[i, j]` = the number of times character `j` directly follows character `i`.

For example, `N[ord('e'), ord('m')] = 32` means the bigram `"em"` appears 32 times in the training data.

### Line 1: `P = (N+1).float()`

This does two things:

- **`N + 1`** — adds 1 to every cell. This is called **Laplace smoothing** (a.k.a. *add-one smoothing*).
- **`.float()`** — converts to floating point, because we're about to divide.

**Why add 1?**

Imagine the training set never contained `"qz"`, so `N[q, z] = 0`. If we used the raw counts directly, the probability of "q followed by z" would be exactly 0. Later, Karpathy uses log-likelihood as the loss — and $\log(0) = -\infty$, which would blow up the entire loss.

So we add 1 to every cell up front. It's like pretending we've seen each possible bigram at least once, guaranteeing every probability is strictly positive. The price: **common combinations get slightly diluted** — but it's a very acceptable tradeoff.

> **The smoothing strength is tunable:** adding 1 is mild smoothing; adding 100 pushes everything toward a uniform distribution (because every cell gets bumped by roughly the same amount).

### Line 2: `P /= P.sum(1, keepdims=True)`

This **row-normalizes** the count matrix, turning every row into a valid probability distribution (each row sums to 1).

Breaking it down:

- **`P.sum(1, keepdims=True)`** — sums along dimension 1, i.e. adds up each row, producing a **27×1** column vector. Each element is "how many times this character occurred *as the preceding character*".
- **`keepdims=True`** — preserves the 2-D shape `(27, 1)` instead of collapsing to 1-D `(27,)`. This is **critical**, otherwise broadcasting will go wrong.
- **`P /= ...`** — divides each row by its own row-sum.

#### Why `dim=1` and not `dim=0`?

- **`sum(0)`** — collapses along dim 0 (row axis) → gives the **column sums** (how often each character appeared as the *following* character).
- **`sum(1)`** — collapses along dim 1 (column axis) → gives the **row sums** (how often each character appeared as the *preceding* character).

We want "given the previous character `i`, what's the probability the next character is `j`", so we normalize **by row** — each row should independently be a distribution. That's exactly `sum(1)`.

#### Why `keepdims=True` matters

```
shape of P                 (27, 27)
P.sum(1, keepdims=True)   (27,  1)   ✅ Broadcasting: each row divided by its own sum
P.sum(1)                  (27,)      ❌ Broadcasting: treated as (1, 27), each column divided by one value!
```

Without `keepdims=True` the result is **completely wrong** — the resulting matrix has each **column** summing to 1 instead of each **row**. Karpathy specifically called out this pitfall in the video.

### A tiny concrete example

Suppose we only have 3 characters `a, b, c`, and the raw `N` is:

```
       a   b   c
   a [ 2,  3,  0 ]
   b [ 1,  0,  4 ]
   c [ 0,  0,  1 ]
```

After adding 1:

```
       a   b   c
   a [ 3,  4,  1 ]   row sum = 8
   b [ 2,  1,  5 ]   row sum = 8
   c [ 1,  1,  2 ]   row sum = 4
```

Divide each row by its own sum:

```
       a       b       c
   a [ 3/8,   4/8,   1/8 ]   = [0.375, 0.500, 0.125]
   b [ 2/8,   1/8,   5/8 ]   = [0.250, 0.125, 0.625]
   c [ 1/4,   1/4,   2/4 ]   = [0.250, 0.250, 0.500]
```

Every row now sums to 1 — a legitimate **conditional probability distribution**. `P[i, j]` now means "given `i`, the probability the next character is `j`" — ready to be used to sample the next character.

### One-line summary

> These two lines turn an "occurrence-count table" into a "conditional-probability table", with `+1` smoothing to avoid zero probabilities. That's the entire "training" of the bigram model — all that's left is to sample from a row of `P` to pick the next character, loop, and you've generated a name.

## Q2: Repackaging bigrams as a supervised-learning dataset

```python
xs, ys = [], []

for w in words[:1]:
    chs = ['.'] + list(w) + ['.']
    for ch1, ch2 in zip(chs, chs[1:]):
        ix1 = stoi[ch1]
        ix2 = stoi[ch2]
        xs.append(ix1)
        ys.append(ix2)

xs = torch.tensor(xs)
ys = torch.tensor(ys)
```

`xs` and `ys` repackage the bigram problem as a **supervised-learning dataset** — `xs` is the "input", `ys` is the "label".

### Walking through an example

Suppose `words[0] = "emma"` (the first name in Karpathy's dataset).

#### Step 1: Add boundary markers

```python
chs = ['.'] + list("emma") + ['.']
# chs = ['.', 'e', 'm', 'm', 'a', '.']
```

A `.` is added at each end, representing "**before the name starts**" and "**after the name ends**".

#### Step 2: Slide a window over adjacent character pairs

`zip(chs, chs[1:])` is the classic "sliding window" trick. It offsets the list by one and pairs adjacent elements:

```
chs       = ['.', 'e', 'm', 'm', 'a', '.']
chs[1:]   = ['e', 'm', 'm', 'a', '.']

zip output:
(., e)
(e, m)
(m, m)
(m, a)
(a, .)
```

Each pair `(ch1, ch2)` means: "**in the training data, `ch1` is followed by `ch2`**".

#### Step 3: Convert to integer IDs and collect separately

```python
ix1 = stoi[ch1]   # integer ID of ch1
ix2 = stoi[ch2]   # integer ID of ch2
xs.append(ix1)    # goes into the "input" column
ys.append(ix2)    # goes into the "label" column
```

Assume `stoi = {'.': 0, 'a': 1, ..., 'e': 5, ..., 'm': 13, ...}`. We end up with:

| `i` | `ch1 → ch2` | `xs[i]` | `ys[i]` |
|-----|-------------|---------|---------|
| 0   | `.` → `e`   | 0       | 5       |
| 1   | `e` → `m`   | 5       | 13      |
| 2   | `m` → `m`   | 13      | 13      |
| 3   | `m` → `a`   | 13      | 1       |
| 4   | `a` → `.`   | 1       | 0       |

As tensors:

```python
xs = tensor([ 0,  5, 13, 13,  1])
ys = tensor([ 5, 13, 13,  1,  0])
```

### Meaning: supervised-learning training samples

Read this as **5 training samples**:

```
sample 0:  input = .  →  correct answer = e
sample 1:  input = e  →  correct answer = m
sample 2:  input = m  →  correct answer = m
sample 3:  input = m  →  correct answer = a
sample 4:  input = a  →  correct answer = .
```

In other words: "**when we see character `xs[i]`, the next character should be `ys[i]`**".

### Comparison with the previous lecture

- **Last lecture's "training" was counting**: `N[i, j]` records how many times `i → j` occurred.
- **This lecture's "training" reformulates it as a neural net** — and neural nets need input as a list of `(input, label)` samples, not as a count table.

So Karpathy expresses the same information in a different form:

| Perspective | Representation |
|-------------|----------------|
| **Statistical view** (last lecture) | "Character `i` is followed by `j` a total of `N[i, j]` times" |
| **Sample view** (this lecture)      | "Here's a pile of samples; each one says `xs[i]` is followed by `ys[i]`" |

**Both representations carry exactly the same information.** For instance, `"m → m"` corresponds to `N[13, 13] += 1` in the statistical view, and one occurrence of `(xs=13, ys=13)` in the sample view. If you re-count all the samples generated this lecture, you'd reconstruct `N` exactly.

### How the neural network uses this

1. Feed `xs` into a neural net (the simplest version is just a **27×27 weight matrix `W`**).
2. The net outputs a predicted probability distribution over the next character.
3. Use `ys` as the "correct answer" and compute the **loss (negative log-likelihood)**.
4. Backpropagate and update `W`.
5. The final learned `W` will be **essentially equivalent** to the normalized `P` from the previous lecture.
